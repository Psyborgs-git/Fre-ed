## Every token looks at every other token

In a standard neural network, each layer's output depends only on its input.
There's no built-in way for position 5 in a sequence to "ask" position 2 what it knows.

**Self-attention** solves this. For each token in a sequence, it computes a weighted
combination of *all* tokens' values — where the weights are learned, query-based scores.
The result: every output token is contextualised by the full sequence.

The 3D scene maps the process spatially:

| 3D object | Meaning |
|---|---|
| Violet spheres (left) | Input token embeddings $x_1 \ldots x_6$ |
| Cyan spheres (Q column) | Query projections $q_i = x_i W^Q$ |
| Amber spheres (K column) | Key projections $k_i = x_i W^K$ |
| Violet spheres (V column) | Value projections $v_i = x_i W^V$ |
| Cyan arcs (Q·K lines) | Dot-product similarity scores $q_i \cdot k_j$ |
| Coloured matrix (centre) | Softmax attention weights $\alpha_{ij}$ |
| Amber aggregation lines | Weighted sum of V projections |
| Cyan spheres (right) | Context-enriched output tokens $o_i$ |

## The Q, K, V decomposition

Self-attention decomposes each input embedding into three roles via three learned
weight matrices $W^Q$, $W^K$, $W^V \in \mathbb{R}^{d \times d_k}$:

$$
q_i = x_i W^Q, \quad k_i = x_i W^K, \quad v_i = x_i W^V
$$

- **Query** $q_i$ — "What information am I looking for?"
- **Key** $k_j$ — "What information do I have?"
- **Value** $v_j$ — "What information do I provide if selected?"

The query from token $i$ is compared against the key from every token $j$ via a
dot product. Tokens with similar queries and keys score highly and attend heavily
to each other's values.

## Scaled dot-product attention

The full formula for one head of self-attention is:

$$
\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{Q K^\top}{\sqrt{d_k}}\right) V
$$

where $Q \in \mathbb{R}^{n \times d_k}$, $K \in \mathbb{R}^{n \times d_k}$,
$V \in \mathbb{R}^{n \times d_v}$, and $n$ is the sequence length.

**Step 1 — Compute all pairwise scores:**

$$
S_{ij} = \frac{q_i \cdot k_j}{\sqrt{d_k}}
$$

The $\sqrt{d_k}$ scaling prevents the dot products from growing too large in magnitude,
which would push the softmax into flat saturation regions and kill gradients.

**Step 2 — Softmax to get attention weights:**

$$
\alpha_{ij} = \frac{\exp(S_{ij})}{\sum_{k=1}^{n} \exp(S_{ik})}
$$

Each row of the attention matrix sums to 1. High-scoring pairs get most of the weight;
low-scoring pairs are effectively ignored.

**Step 3 — Weighted sum of values:**

$$
o_i = \sum_{j=1}^{n} \alpha_{ij}\, v_j
$$

The output token $o_i$ is a mixture of all values, blended by how relevant each token $j$
is to the query of token $i$.

<CodeBlock language="python">
{`import numpy as np

def scaled_dot_product_attention(Q, K, V):
    """
    Q: (n, d_k)   — query matrix
    K: (n, d_k)   — key matrix
    V: (n, d_v)   — value matrix
    Returns context vectors of shape (n, d_v).
    """
    d_k = Q.shape[-1]
    # Pairwise dot products: (n, n)
    scores = Q @ K.T / np.sqrt(d_k)

    # Softmax over keys dimension (axis=-1)
    scores = scores - scores.max(axis=-1, keepdims=True)   # numerical stability
    weights = np.exp(scores)
    weights = weights / weights.sum(axis=-1, keepdims=True) # (n, n)

    # Weighted sum of values: (n, d_v)
    out = weights @ V
    return out, weights

# Example: 4 tokens, d_k = d_v = 8
np.random.seed(0)
n, d_k = 4, 8
W_Q = np.random.randn(8, d_k) * 0.1
W_K = np.random.randn(8, d_k) * 0.1
W_V = np.random.randn(8, d_k) * 0.1

X = np.random.randn(n, 8)          # 4 tokens, embedding dim 8
Q = X @ W_Q                        # (4, 8)
K = X @ W_K
V = X @ W_V

context, attn_weights = scaled_dot_product_attention(Q, K, V)
print("Context shape:", context.shape)          # (4, 8)
print("Attention weights:\\n", attn_weights.round(3))`}
</CodeBlock>

## Multi-head attention

A single attention head learns one type of relationship. **Multi-head attention** runs
$h$ heads in parallel, each with its own $W^Q_i$, $W^K_i$, $W^V_i$ projections:

$$
\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h)\, W^O
$$

$$
\text{head}_i = \text{Attention}(Q W^Q_i,\; K W^K_i,\; V W^V_i)
$$

Where $W^O \in \mathbb{R}^{h d_v \times d_{\text{model}}}$ is a learned output projection.

Each head can specialise:
- One head might capture **syntactic** relationships (subject → verb)
- Another might capture **semantic** co-reference ("it" → "the cat")
- A third might capture **positional** proximity

<Callout type="tip">
GPT-2 (117M params) uses 12 heads of dimension 64 each. GPT-4 reportedly uses 128 heads.
More heads allow finer-grained contextual specialisation but increase computation quadratically
with sequence length — the core bottleneck of the transformer architecture.
</Callout>

<CodeBlock language="python">
{`import torch
import torch.nn as nn

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model=512, n_heads=8):
        super().__init__()
        assert d_model % n_heads == 0
        self.d_k = d_model // n_heads
        self.n_heads = n_heads

        self.W_Q = nn.Linear(d_model, d_model)  # projects to all heads at once
        self.W_K = nn.Linear(d_model, d_model)
        self.W_V = nn.Linear(d_model, d_model)
        self.W_O = nn.Linear(d_model, d_model)

    def forward(self, x, mask=None):
        B, n, d = x.shape

        # Project and split into heads: (B, n, d_model) → (B, n_heads, n, d_k)
        def project_and_split(linear):
            out = linear(x).view(B, n, self.n_heads, self.d_k)
            return out.transpose(1, 2)   # (B, n_heads, n, d_k)

        Q = project_and_split(self.W_Q)
        K = project_and_split(self.W_K)
        V = project_and_split(self.W_V)

        # Scaled dot-product attention per head
        scores = (Q @ K.transpose(-2, -1)) / (self.d_k ** 0.5)  # (B, h, n, n)
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))
        weights = scores.softmax(dim=-1)   # (B, h, n, n)
        context = weights @ V              # (B, h, n, d_k)

        # Concatenate heads and project
        context = context.transpose(1, 2).contiguous().view(B, n, d)  # (B, n, d_model)
        return self.W_O(context), weights

# Test
mha = MultiHeadAttention(d_model=512, n_heads=8)
x = torch.randn(2, 10, 512)   # batch=2, seq_len=10, d_model=512
out, weights = mha(x)
print("Output:", out.shape)            # (2, 10, 512)
print("Attention weights:", weights.shape)  # (2, 8, 10, 10)`}
</CodeBlock>

## Causal (masked) self-attention

In a **decoder** (as in GPT), each token can only attend to *previous* tokens —
it cannot look ahead at future positions. This is enforced by an additive mask
that sets future scores to $-\infty$ before the softmax:

$$
S_{ij} = \begin{cases}
\dfrac{q_i \cdot k_j}{\sqrt{d_k}} & j \leq i \\[6pt]
-\infty & j > i
\end{cases}
$$

After softmax, $-\infty$ scores become exactly 0, so future tokens are completely
excluded from the context vector. The causal mask is a lower-triangular matrix of ones.

## Complexity: the quadratic bottleneck

The attention computation requires $O(n^2 d_k)$ operations for a sequence of length $n$.

| Operation | Complexity |
|---|---|
| QKV projections | $O(n \, d^2)$ |
| Score computation $QK^\top$ | $O(n^2 d_k)$ — quadratic! |
| Value aggregation | $O(n^2 d_v)$ |

For $n = 1{,}000$ tokens, the score matrix has $10^6$ entries.
For $n = 100{,}000$ (book-length context), it has $10^{10}$ — infeasible with dense attention.

This is why modern long-context models use approximate or sparse attention:
**FlashAttention** (fused GPU kernel), **sliding window attention** (Longformer),
**linear attention** (Performer) — all attempts to tame the $O(n^2)$ wall.

<Callout type="warning">
The quadratic complexity in sequence length is the fundamental scaling challenge for
transformer models. It is why GPT-2 was limited to 1,024 tokens and why modern
architectures invest heavily in attention approximations.
</Callout>

## Positional information and attention

A key subtlety: vanilla self-attention is **permutation-equivariant**. Rearranging
tokens produces the same output, just rearranged — the attention mechanism has no
notion of order.

Position information is injected via **positional encodings** added to the input
embeddings before computing Q, K, V:

$$
x'_i = x_i + PE_i
$$

The original transformer used sinusoidal encodings:

$$
PE_{i, 2k} = \sin\!\left(\frac{i}{10000^{2k/d}}\right), \quad
PE_{i, 2k+1} = \cos\!\left(\frac{i}{10000^{2k/d}}\right)
$$

Modern LLMs (LLaMA, GPT-4) use **Rotary Position Embeddings (RoPE)**, which encode
relative position directly into the Q·K dot product, enabling extrapolation to
longer sequences than seen during training.

## Summary

$$
\underbrace{x_1 \ldots x_n}_{\text{input tokens}}
\xrightarrow{\;W^Q, W^K, W^V\;}
\underbrace{Q, K, V}_{\text{projections}}
\xrightarrow{\;\text{softmax}(QK^\top/\sqrt{d_k})\;}
\underbrace{\alpha_{ij}}_{\text{weights}}
\xrightarrow{\;\alpha V\;}
\underbrace{o_1 \ldots o_n}_{\text{context vectors}}
$$

- Each token creates a **Query** (what I want), a **Key** (what I offer), and a **Value** (what I provide).
- Attention scores = scaled dot products between all Q-K pairs; softmax turns scores into weights.
- Each output token is a weighted blend of all value vectors.
- Multi-head attention runs this process in parallel with different projections, capturing diverse relationships.
- Causal masking restricts decoders to attend only to past tokens.
