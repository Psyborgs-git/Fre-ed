## Mean-variance framework

In 1952, Harry Markowitz showed that rational investors should care about two things: **expected return** and **variance** (risk). Given $N$ assets with expected return vector $\boldsymbol{\mu}$ and covariance matrix $\Sigma$, a portfolio with weight vector $\mathbf{w}$ (summing to 1) has:

$$
\mu_p = \mathbf{w}^\top \boldsymbol{\mu}, \qquad \sigma_p^2 = \mathbf{w}^\top \Sigma \, \mathbf{w}
$$

The key insight: portfolio variance depends not only on individual asset variances (diagonal of $\Sigma$) but on all pairwise covariances (off-diagonal). Combining assets with low or negative covariances reduces $\sigma_p^2$ below the weighted average of individual variances — **the diversification benefit**.

<CodeBlock language="python">{`import numpy as np

def portfolio_stats(weights: np.ndarray,
                    mu: np.ndarray,
                    cov: np.ndarray,
                    rf: float = 0.05) -> dict:
    """Compute return, vol, and Sharpe ratio for a portfolio."""
    port_return = weights @ mu
    port_var    = weights @ cov @ weights
    port_vol    = np.sqrt(port_var)
    sharpe      = (port_return - rf) / port_vol
    return {"return": port_return, "vol": port_vol, "sharpe": sharpe}

def random_portfolios(mu: np.ndarray, cov: np.ndarray,
                      n_portfolios: int = 5000,
                      rf: float = 0.05) -> list[dict]:
    """Monte Carlo simulation of random portfolio weights."""
    n = len(mu)
    results = []
    for _ in range(n_portfolios):
        w = np.random.dirichlet(np.ones(n))  # random weights summing to 1
        stats = portfolio_stats(w, mu, cov, rf)
        stats["weights"] = w
        results.append(stats)
    return results

## The efficient frontier

The **efficient frontier** is the set of portfolios with the highest expected return for each level of portfolio risk. Every other portfolio is **dominated** — you can do better for the same risk, or equivalently take less risk for the same return.

Mathematically, the efficient frontier solves:

$$
\min_{\mathbf{w}} \; \mathbf{w}^\top \Sigma \, \mathbf{w} \quad \text{s.t.} \quad \mathbf{w}^\top \boldsymbol{\mu} = \mu^*, \quad \mathbf{w}^\top \mathbf{1} = 1, \quad \mathbf{w} \geq 0
$$

for each target return $\mu^*$. The solution traces out a parabola in $(\sigma, \mu)$ space. In the scene, the bright **cyan curve** highlights the pareto-efficient subset.

The **two-fund separation theorem** says every efficient portfolio is a linear combination of just two portfolios: the minimum-variance portfolio and any other efficient portfolio.

<CodeBlock language="python">{`import numpy as np
from scipy.optimize import minimize

def efficient_frontier(mu: np.ndarray, cov: np.ndarray,
                       n_points: int = 100,
                       rf: float = 0.05) -> list[dict]:
    """
    Trace the efficient frontier by minimising variance for target returns
    ranging from min-variance to max-return portfolio.
    """
    n = len(mu)
    constraints = [{"type": "eq", "fun": lambda w: w.sum() - 1}]
    bounds = [(0, 1)] * n  # long-only

    # Min-variance portfolio (no return target)
    def port_variance(w): return w @ cov @ w
    res_mv = minimize(port_variance, x0=np.ones(n)/n,
                      method='SLSQP', bounds=bounds, constraints=constraints)
    mu_min  = res_mv.x @ mu
    mu_max  = mu.max()

    frontier = []
    for mu_target in np.linspace(mu_min, mu_max, n_points):
        cons = constraints + [{"type": "eq", "fun": lambda w, m=mu_target: w @ mu - m}]
        res  = minimize(port_variance, x0=res_mv.x,
                        method='SLSQP', bounds=bounds, constraints=cons)
        if res.success:
            stats = portfolio_stats(res.x, mu, cov, rf)
            stats["weights"] = res.x
            frontier.append(stats)
    return frontier

frontier = efficient_frontier(mu, cov, n_points=100)
print(f"Frontier spans vol: {min(p['vol'] for p in frontier):.3f} "
      f"— {max(p['vol'] for p in frontier):.3f}")
`}</CodeBlock>

## Min-variance and max-Sharpe portfolios

Two special points on the efficient frontier deserve attention:

**Minimum-variance portfolio (MVP)**: the leftmost point on the frontier — the portfolio with the smallest possible variance regardless of return. It is found by solving:

$$
\mathbf{w}^* = \frac{\Sigma^{-1} \mathbf{1}}{\mathbf{1}^\top \Sigma^{-1} \mathbf{1}}
$$

**Maximum-Sharpe portfolio (MSP / tangency portfolio)**: the portfolio on the frontier with the highest Sharpe ratio — the point where a line from the risk-free rate is tangent to the frontier. When investors can borrow/lend at $r_f$, every rational investor holds the MSP levered or de-levered to their risk preference:

$$
\mathbf{w}^* = \frac{\Sigma^{-1}(\boldsymbol{\mu} - r_f \mathbf{1})}{\mathbf{1}^\top \Sigma^{-1}(\boldsymbol{\mu} - r_f \mathbf{1})}
$$

<CodeBlock language="python">{`import numpy as np
from scipy.optimize import minimize

def min_variance_portfolio(mu: np.ndarray, cov: np.ndarray) -> dict:
    n = len(mu)
    def port_vol(w): return np.sqrt(w @ cov @ w)
    cons   = [{"type": "eq", "fun": lambda w: w.sum() - 1}]
    bounds = [(0, 1)] * n
    res = minimize(port_vol, x0=np.ones(n)/n,
                   method='SLSQP', bounds=bounds, constraints=cons)
    return {"weights": res.x, **portfolio_stats(res.x, mu, cov)}

def max_sharpe_portfolio(mu: np.ndarray, cov: np.ndarray,
                         rf: float = 0.05) -> dict:
    n = len(mu)
    def neg_sharpe(w):
        ret = w @ mu
        vol = np.sqrt(w @ cov @ w)
        return -(ret - rf) / vol if vol > 1e-10 else 0.0
    cons   = [{"type": "eq", "fun": lambda w: w.sum() - 1}]
    bounds = [(0, 1)] * n
    res = minimize(neg_sharpe, x0=np.ones(n)/n,
                   method='SLSQP', bounds=bounds, constraints=cons)
    return {"weights": res.x, **portfolio_stats(res.x, mu, cov, rf)}

mvp = min_variance_portfolio(mu, cov)
msp = max_sharpe_portfolio(mu, cov, rf=0.05)
print(f"MVP  — vol: {mvp['vol']:.3f}, return: {mvp['return']:.3f}")
print(f"MSP  — vol: {msp['vol']:.3f}, return: {msp['return']:.3f}, Sharpe: {msp['sharpe']:.3f}")
`}</CodeBlock>

## Beyond mean-variance optimisation

MVO is elegant but fragile in practice:

- **Estimation error**: sample means are extremely noisy. Small errors in $\boldsymbol{\mu}$ produce wildly different frontier portfolios — the optimiser **maximises estimation error**.
- **Corner solutions**: unconstrained MVO often places 100% in one or two assets with the highest estimated Sharpe.
- **Non-normality**: fat tails, skewness, and co-crash risk are ignored.

**Practical remedies:**

| Issue | Remedy |
|-------|--------|
| Noisy mean estimates | Black-Litterman model (blend prior with views) |
| Noisy covariance | Ledoit-Wolf shrinkage, factor models |
| Concentration | Max-diversification, risk-parity constraints |
| Fat tails | CVaR/ES optimisation instead of variance |

The **Black-Litterman (BL)** model provides a Bayesian framework: start with market-cap-implied returns as a prior, blend in your own views with confidence weights, and solve MVO on the posterior expected returns. This produces stable, diversified portfolios without corner solutions.

$$
\boldsymbol{\mu}_{\text{BL}} = \left[(\tau\Sigma)^{-1} + P^\top \Omega^{-1} P\right]^{-1} \left[(\tau\Sigma)^{-1}\boldsymbol{\pi} + P^\top \Omega^{-1}\mathbf{Q}\right]
$$

where $\boldsymbol{\pi}$ is the CAPM equilibrium return vector, $P$ is the view matrix, $\mathbf{Q}$ is the view return vector, $\Omega$ is view uncertainty, and $\tau$ is a scalar.

<Callout type="warning">
Never deploy an MVO strategy without out-of-sample testing and stability analysis. Optimal weights from in-sample data frequently perform worse than a naive equal-weight portfolio out-of-sample due to estimation error. Use rolling-window re-optimisation with regularisation constraints ($w_{\max} \leq 20\%$ per asset) at minimum.
</Callout>
