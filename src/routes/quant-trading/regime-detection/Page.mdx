## Market regimes

Markets do not behave the same way over all time periods. Practitioners distinguish between at least three broad **regimes**:

| Regime  | Colour | Characteristics |
|---------|--------|-----------------|
| **Bull** | Green  | Rising prices, positive drift, low mean-reversion |
| **Bear** | Red    | Falling prices, negative drift, elevated volatility |
| **Ranging** | Amber | Sideways, high mean-reversion, volatility clusters |

The scene renders a synthetic price series coloured by its regime. As you scroll, the price builds left-to-right through a bull phase, a crash, a ranging consolidation, and then another bull phase. After 60% scroll, three spheres appear in a triangle — the **Hidden Markov Model** state diagram, with curved arrows showing transition probabilities between regimes.

Regime-awareness matters because:
- A **trend-following** strategy thrives in bull and bear regimes but loses money in ranging markets.
- A **mean-reversion** strategy profits in ranging regimes but gets destroyed in strong trends.
- **Volatility strategies** need separate sizing for each regime's variance level.

<CodeBlock language="python">{`import numpy as np
import pandas as pd

def classify_regime_simple(prices: pd.Series,
                            fast: int = 50,
                            slow: int = 200,
                            vol_window: int = 20) -> pd.Series:
    """
    Simple moving-average crossover + volatility regime classifier.
    Returns: 0 = bull, 1 = bear, 2 = ranging.
    """
    fast_ma = prices.rolling(fast).mean()
    slow_ma = prices.rolling(slow).mean()
    vol     = prices.pct_change().rolling(vol_window).std()
    vol_med = vol.rolling(252).median()

    regime = pd.Series(index=prices.index, dtype=int)
    trend_up   = fast_ma > slow_ma
    high_vol   = vol > vol_med * 1.5

    regime[trend_up  & ~high_vol] = 0   # bull
    regime[~trend_up & ~high_vol] = 1   # bear
    regime[high_vol]               = 2   # ranging / volatile

    return regime.fillna(2)
`}</CodeBlock>

## Hidden Markov Models

A **Hidden Markov Model (HMM)** assumes that observed returns are generated by an unobserved (hidden) discrete state $s_t \in \{1, \ldots, K\}$ that evolves according to a Markov chain. At each step, the state emits an observation drawn from a state-specific distribution.

**Parameters of a $K$-state HMM:**

$$
\boldsymbol{\pi} = (\pi_1, \ldots, \pi_K) \quad \text{initial state probabilities}
$$

$$
A_{ij} = P(s_{t+1} = j \mid s_t = i) \quad \text{transition matrix},\quad \sum_j A_{ij} = 1
$$

$$
p(r_t \mid s_t = k) = \mathcal{N}(\mu_k, \sigma_k^2) \quad \text{emission distribution per state}
$$

The **Baum-Welch algorithm** (Expectation-Maximisation) fits these parameters to observed return data. At inference time, the **Viterbi algorithm** finds the most likely state sequence, while the **forward-backward algorithm** gives posterior probabilities $P(s_t = k \mid r_{1:T})$.

$$
\gamma_t(k) = P(s_t = k \mid r_{1:T}) = \frac{\alpha_t(k)\,\beta_t(k)}{\sum_{j} \alpha_t(j)\,\beta_t(j)}
$$

<CodeBlock language="python">{`from hmmlearn.hmm import GaussianHMM
import numpy as np
import pandas as pd

def fit_hmm_regimes(prices: pd.Series,
                    n_states: int = 3,
                    n_iter: int = 200) -> pd.DataFrame:
    """
    Fit a Gaussian HMM to log-returns and return regime labels and probabilities.
    """
    log_ret = np.log(prices / prices.shift(1)).dropna().values.reshape(-1, 1)

    model = GaussianHMM(
        n_components=n_states,
        covariance_type="full",
        n_iter=n_iter,
        random_state=42,
    )
    model.fit(log_ret)

    hidden_states = model.predict(log_ret)
    proba         = model.predict_proba(log_ret)

    # Order states by mean return: 0=bear(lowest), 1=ranging, 2=bull(highest)
    order     = np.argsort(model.means_.flatten())
    inv_order = np.argsort(order)
    regime    = inv_order[hidden_states]

    result = pd.DataFrame(
        proba[:, order],
        index=prices.index[1:],
        columns=[f"P(state_{k})" for k in range(n_states)],
    )
    result["regime"] = regime
    result["mu"]     = model.means_[hidden_states].flatten()
    result["sigma"]  = np.sqrt(model.covars_[hidden_states].flatten())
    return result

## Volatility clustering

A key stylised fact of financial markets is **volatility clustering**: large moves tend to be followed by large moves (of either sign). This is captured by the autocorrelation of absolute or squared returns:

$$
\text{Corr}(|r_t|, |r_{t-k}|) > 0 \quad \text{for } k = 1, 2, \ldots, \text{many lags}
$$

The **GARCH(1,1)** model makes this precise:

$$
r_t = \mu + \epsilon_t, \quad \epsilon_t = \sigma_t z_t, \quad z_t \sim \mathcal{N}(0,1)
$$

$$
\sigma_t^2 = \omega + \alpha \epsilon_{t-1}^2 + \beta \sigma_{t-1}^2
$$

Volatility regimes emerge naturally: the persistence parameter $\alpha + \beta$ close to 1 means vol shocks decay slowly, producing long stretches of elevated or depressed volatility.

<CodeBlock language="python">{`from arch import arch_model
import numpy as np
import pandas as pd

def garch_vol_regimes(prices: pd.Series,
                      low_pct: float = 33,
                      high_pct: float = 67) -> pd.Series:
    """
    Fit GARCH(1,1) and classify vol regime as low / medium / high.
    Returns Series: 0=low, 1=medium, 2=high.
    """
    ret = 100 * np.log(prices / prices.shift(1)).dropna()

    model = arch_model(ret, vol="Garch", p=1, q=1, dist="normal", rescale=False)
    res   = model.fit(disp="off")

    cond_vol = res.conditional_volatility.rename("garch_vol")
    lo  = cond_vol.quantile(low_pct  / 100)
    hi  = cond_vol.quantile(high_pct / 100)

    regime = pd.cut(cond_vol, bins=[-np.inf, lo, hi, np.inf],
                    labels=[0, 1, 2]).astype(int)
    return regime

# Example
import yfinance as yf
prices = yf.download("SPY", start="2015-01-01", end="2024-01-01")["Close"]
vol_regime = garch_vol_regimes(prices)
print(vol_regime.value_counts().sort_index())
`}</CodeBlock>

## Regime-conditional strategies

Once regimes are identified, **strategy parameters should adapt to the active regime**:

| Regime     | Trend-following | Mean reversion | Position size |
|------------|----------------|----------------|---------------|
| Bull       | Full long       | Avoid          | 1× vol-target |
| Bear       | Full short      | Avoid          | 0.7× (higher corr with other shorts) |
| Ranging    | Flat / hedge    | Full weight    | 0.8× |

The conditional expected return given regime is:

$$
\mathbb{E}[r_{t+1} \mid s_t = k] = \mu_k
$$

A **regime-switching overlay** allocates to sub-strategies by posterior probability:

$$
w_t^{\text{composite}} = \sum_{k=1}^{K} \gamma_t(k) \cdot w_t^{(k)}
$$

<Callout type="warning">
Regime models suffer from **look-ahead bias** when developed on the full sample. Always use only information available at time $t$ (real-time filter probabilities, not smoothed) in backtests. HMM smoothing uses future data — use the forward-pass only for live trading.
</Callout>
